How It Works

From a single RFID scan to a full narrated, guided tour — five precise steps that power the entire Itihaas Sarthi experience.

The Process

From activation to narration

Every interaction with Itihaas Sarthi follows the same seamless flow — designed so visitors spend zero time figuring out the technology and 100% of their attention on the history around them.

01
RFID ActivationAn authorised RFID tag wakes the robot. This ensures only approved users activate the system and provides a secure, contactless startup — no buttons, no passwords.
02
Location DetectionThe GPS module triangulates position while the onboard camera scans the environment. Together they confirm the exact heritage site and load the matching knowledge set.
03
Facts RetrievedHistorical facts are pulled from onboard SD card memory — no internet needed. Content covers architecture, rulers, dates, legends, and cultural significance of each monument.
04
Audio NarrationThe text-to-speech module converts stored facts into natural spoken audio through the onboard speaker. Language is selected with one button — Hindi, English, or regional. Volume is adjustable with tactile controls accessible to all users.
05
Guided MovementServo-driven wheels move the robot forward while ultrasonic sensors scan ahead for walls, people, and monuments. Cliff sensors detect drops at stairs. Vibration motors alert the user 2–3 seconds before every turn or stop — keeping the experience safe and predictable.

Detailed Flow

Step-by-step in detail

User scans RFID tag

The RFID sensor reads the tag and sends a signal to the Arduino Uno microcontroller. The robot powers up from standby, runs a quick sensor self-check, and extends the retractable handle if a touch sensor detects a user waiting.

GPS confirms site location

The GPS module reads current coordinates and matches them against preloaded site data on the SD card. If an optional NFC or QR reader is installed, monument-specific triggers are also checked for finer granularity within large sites.

Content loaded from SD card

Facts for the matched site are read from the Micro SD card via the SD card module. Content is structured in blocks — introduction, architecture, history, legends — so the robot can narrate progressively as the visitor moves through the site.

TTS converts text to speech

The TTS module processes each text block and outputs audio through the speaker. The visitor can pause narration, skip ahead, or repeat using voice commands (if enabled) or physical buttons on the robot's body.

Robot moves and guides

The motor driver controls all four wheels, adjusting speed based on terrain. Obstacle data from the ultrasonic sensor is processed every 100ms. The vibration motor triggers 2.5 seconds before each directional change, giving the visitor time to prepare their next step.
